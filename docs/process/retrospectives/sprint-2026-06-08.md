## What went well?

- **Two-mode skill delivered and validated end-to-end.** The new `dry-run | code` contract was
  proven by a real 15-phase dry-run of SPEC-LGS-001 (phase-executor subagent per phase, real
  `make` evidence — 945 unit tests / 86.56% coverage). The dry-run invariant held: only the
  gitignored `reports/` sandbox was written.
- **Review-before-merge caught real defects every time.** The two-mode change review found 7 issues
  pre-merge; the F7 guard review found **3 genuine bypasses + a fail-closed crash** — shipping it
  unreviewed would have meant a security guard with a `git -C … push` hole (worse than no guard).
- **Safety moved from prose to the harness layer.** F7's `PreToolUse` guard makes "stop at every
  human gate" real — `deny` for subagents, `ask` in the main session — backed by `make
verify-f7-hook` (29 cases) as a regression gate.
- **Disciplined finding→issue→fix→trace loop.** Every finding was filed, fixed, and recorded in
  `docs/sdlc/deliver-dryrun-findings.md`; issues #129–133, #138, #141 all closed with merge refs.
- **Branch hygiene held throughout** — every change went through a PR; `develop` re-synced to
  `main` after each merge; no direct pushes to protected branches.

## What slowed us down?

- **Local lint ≠ CI lint (F9).** `make lint-python` only checked `src/ tests/` and lacked
  `ruff format --check`, while CI runs `ruff check .` + `ruff format --check .`. The new
  `.claude/` Python passed locally but failed CI **twice** on #140 — avoidable round-trips. Fixed
  in #142 (local lint is now authoritative).
- **Stale `uv.lock` (F8)** had to be manually reverted twice (in #128 and #136) before being
  properly refreshed in #139 — phantom diffs from every `uv run`.
- **Auto-merge eagerness.** #128 auto-merged at `8c9b912` before a follow-up doc commit could ride
  along, stranding it on a dead branch and requiring a cherry-pick + #134. Eager auto-merge of
  docs/`.md` PRs is a sharp edge to be aware of.
- **phase-gates.yaml drift (F2–F4)** meant the workflow's own gate definitions were partly
  inaccurate (DoR count, a non-existent `smoke-test.yml`, manual-CHANGELOG vs release-please).
